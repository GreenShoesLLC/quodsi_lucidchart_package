# 02. Local Startup

Detailed step-by-step guide for starting all services in the correct order.

## Architecture Overview

The Quodsi system consists of 4 components that run simultaneously:

```
Terminal 1: Shared Library     → Build TypeScript types/validation (one-time)
Terminal 2: Azure Function      → Backend API for simulation (port 7071)
Terminal 3: React App           → UI dev server (port 3000)
Terminal 4: Extension Test      → LucidChart extension in test mode
```

**Why 4 terminals?**
- Each component runs independently
- Services need to communicate (extension ↔ React ↔ Function)
- Hot reload requires separate processes
- Build order matters (shared library first)

---

## Build Order Matters

**Critical:** The shared library (`@quodsi/shared`) must be built FIRST because:
- Contains TypeScript types used by all components
- Contains validation logic
- Contains serialization/deserialization code
- Extension and data connector import from it

**Dependency Chain:**
```
@quodsi/shared (shared library)
    ↓
    ├─→ Extension (TypeScript)
    ├─→ React App (TypeScript)
    └─→ Data Connector (TypeScript)
```

---

## Terminal 1: Shared Library

### Purpose

The `@quodsi/shared` workspace contains:
- **TypeScript Types:** ModelDefinition, Activity, Resource, etc.
- **Validation:** ModelValidationService
- **Serialization:** ModelSerializerFactory
- **Messaging:** Envelope types, message protocol
- **Enums:** SimulationStatus, ComponentType, etc.

These are used by:
- Extension (model management, validation)
- React App (UI types, validation display)
- Data Connector (model serialization, batch submission)

### Startup Commands

**From repository root**, copy and paste into Terminal 1:

```bash
npm install
npm run build -w @quodsi/shared
```

### Expected Output

```
> @quodsi/shared@1.0.0 build
> tsc

[Build complete message]
```

### What It Does

1. `npm install` - Installs/updates all dependencies for the monorepo
2. `npm run build -w @quodsi/shared` - Builds just the shared workspace
   - Compiles TypeScript to JavaScript
   - Outputs to `shared/dist/`
   - Generates type declarations (`.d.ts`)

### Build Output

```
shared/
├── src/
│   └── [TypeScript source files]
└── dist/                    ← Generated by build
    ├── index.js
    ├── index.d.ts
    ├── types/
    ├── validation/
    └── ...
```

### When to Rebuild

Rebuild shared library when:
- You modify any file in `shared/src/`
- You pull changes that affect shared library
- Other services show "Cannot find module '@quodsi/shared'" errors

**Quick rebuild:**
```bash
npm run build -w @quodsi/shared
```

## Terminal 2: Azure Function (Data Connector)

### Purpose

The data connector is an Azure Function that:
- Receives model definition from extension
- Uploads model JSON to Azure Blob Storage
- Submits Azure Batch jobs for simulation execution
- Returns job ID and status to extension

**Role in simulation flow:** See `editorextensions/quodsi_editor_extension/_docs/architecture/messaging/simulation/04_data_connector_integration.md`

### Startup Commands

**From repository root**, copy and paste into Terminal 2:

```bash
cd dataconnectors/quodsi_data_connector_lucidchart_v2
npm run build
func start --debug --verbose
```

### Expected Output

```
Azure Functions Core Tools
Core Tools Version:       4.x.xxxx
Function Runtime Version: 4.x.x.xxxxx

Functions:

  SaveAndSubmitSimulation: [POST] http://localhost:7071/api/simulation/save-and-submit

  [Other function endpoints...]

For detailed output, run func with --verbose flag.
```

### What It Does

1. `cd dataconnectors/quodsi_data_connector_lucidchart_v2` - Navigate to function app
2. `npm run build` - Compile TypeScript to JavaScript
3. `func start --debug --verbose` - Start Azure Functions runtime
   - `--debug` - Enable debugging
   - `--verbose` - Show detailed logs

### Port and Endpoints

**Base URL:** `http://localhost:7071`

**Endpoints:**
- `POST /api/simulation/save-and-submit` - Submit simulation job
- (Other endpoints as configured)

### Configuration

Uses `local.settings.json` for:
- Azure Storage connection string
- Azure Batch credentials
- Environment variables

**If missing:** Function starts but simulation submission fails

### When to Restart

Restart when:
- You modify TypeScript files in `dataconnectors/.../src/`
- You change `local.settings.json`
- You update shared library (rebuild function too)

**Quick restart:**
1. Stop function (Ctrl+C in Terminal 2)
2. Rebuild and restart:
   ```bash
   npm run build
   func start --debug --verbose
   ```

## Terminal 3: React App (UI)

### Purpose

The React app provides the model editor panel embedded in LucidChart:
- Model validation display
- Element editors (Activity, Resource, etc.)
- Simulation controls ("Run Simulation" button)
- Status updates and progress display

**Messaging:** Communicates with extension via postMessage protocol

### Startup Commands

**From repository root**, copy and paste into Terminal 3:

```bash
cd editorextensions/quodsi_editor_extension/quodsim-react
npx react-scripts start
```

### Expected Output

```
Compiled successfully!

You can now view quodsim-react in the browser.

  Local:            http://localhost:3000
  On Your Network:  http://192.168.x.x:3000

Note that the development build is not optimized.
To create a production build, use npm run build.

webpack compiled successfully
```

**Browser:** May auto-open to `http://localhost:3000`

### What It Does

1. `cd editorextensions/.../quodsim-react` - Navigate to React app
2. `npx react-scripts start` - Start React development server
   - Compiles React/TypeScript
   - Enables hot reload
   - Serves on port 3000
   - Opens browser automatically

### Port

**Development Server:** `http://localhost:3000`

**Note:** You won't interact with this URL directly. The React app is embedded in the LucidChart extension panel (Terminal 4).

### Hot Reload

**Enabled:** Changes to React components automatically reload in browser

**When it works:**
- Modify any `.tsx`, `.ts`, `.css` file in `quodsim-react/src/`
- Save file
- Browser updates automatically (< 2 seconds)

**When to manual reload:**
- Changes to `public/` folder
- Changes to configuration files
- Changes to environment variables

### When to Restart

Restart when:
- Port 3000 conflict
- Hot reload stops working
- Build errors that won't clear

**Quick restart:**
1. Stop server (Ctrl+C in Terminal 3)
2. Restart:
   ```bash
   npx react-scripts start
   ```

## Terminal 4: Extension Test Mode

### Purpose

Launches the LucidChart extension in a test environment where you can:
- Create/edit Quodsi models
- Test model validation
- Run simulations (with Azure configured)
- Debug extension-React communication
- Test all features without publishing

**Development mode:** Changes to extension code require restart (no hot reload)

### Startup Commands

**From repository root**, copy and paste into Terminal 4:

```bash
npx lucid-package@latest test-editor-extension quodsi_editor_extension
```

### Expected Output

```
Executing onWatchRun build scripts
Listening at http://localhost:9900/extension.js
Listening at http://localhost:9901/shapeLibraries

webpack 5.92.1 compiled successfully in 84223 ms
```

**Note:** Browser does NOT auto-open - you must manually navigate to `https://lucid.app`

### What It Does

1. `npx lucid-package@latest` - Uses Lucid's extension SDK CLI
2. `test-editor-extension` - Command to launch test environment
3. `quodsi_editor_extension` - Extension directory name

**Behind the scenes:**
- Watches extension source files for changes
- Serves extension at `http://localhost:9900/extension.js`
- Serves shape libraries at `http://localhost:9901/shapeLibraries`
- Connects to your React dev server (port 3000)

### How to Use

After extension compiles successfully:

1. **Open LucidChart** - Navigate to `https://lucid.app` in your browser
2. **Create a new document** or open existing
3. **Open extension panel:**
   - Look for "Quodsi" in left panel
   - Or: More Tools → Quodsi
4. **Start modeling:**
   - Drag Quodsi shapes from shape library
   - Configure elements in panel
   - Validate model
   - Run simulation

### Extension Panel

The panel loads your React app from `http://localhost:3000`:
- Real-time updates when you modify React code
- Communication with extension via postMessage
- All features available (validation, simulation, etc.)

### When to Restart

Restart when:
- You modify extension TypeScript files (`src/` in extension directory)
- You modify manifest files (`manifest_dev.json`)
- Extension panel not loading
- Need to clear extension state

**Quick restart:**
1. Stop command (Ctrl+C in Terminal 4)
2. Restart:
   ```bash
   npx lucid-package@latest test-editor-extension quodsi_editor_extension
   ```

**Note:** React changes (Terminal 3) hot reload automatically - no need to restart Terminal 4

## Development Workflow

### Making Changes

**Shared Library Changes:**
1. Modify file in `shared/src/`
2. Rebuild: `npm run build -w @quodsi/shared` (Terminal 1)
3. Restart dependent services (Terminal 2, Terminal 4)

**Extension Changes:**
1. Modify file in `editorextensions/quodsi_editor_extension/src/`
2. Restart Terminal 4 (extension test)
3. Refresh LucidChart browser

**React Changes:**
1. Modify file in `quodsim-react/src/`
2. Hot reload automatic
3. Panel updates in 1-2 seconds

**Data Connector Changes:**
1. Modify file in `dataconnectors/.../src/`
2. Stop Terminal 2 (Ctrl+C)
3. Rebuild and restart:
   ```bash
   npm run build
   func start --debug --verbose
   ```

### Stopping Services

**Stop all:**
- Press Ctrl+C in each terminal
- Close browser tabs

**Stop individual:**
- Ctrl+C in specific terminal
- Service stops, others keep running

### Restarting After Code Pull

When you pull changes from Git:

1. **Update dependencies:**
   ```bash
   npm install
   ```

2. **Rebuild shared library:**
   ```bash
   npm run build -w @quodsi/shared
   ```

3. **Restart all services** (Terminals 2-4)

## Troubleshooting

See [04. Troubleshooting](./04_troubleshooting.md) for complete guide.

**Quick fixes:**

- **Port conflicts:** Kill processes using ports 3000 or 7071
- **Module not found:** Rebuild shared library
- **Extension not loading:** Hard refresh browser (Ctrl+Shift+R)
- **Function errors:** Check `local.settings.json` exists
