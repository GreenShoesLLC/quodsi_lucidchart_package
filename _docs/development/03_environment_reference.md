# 03. Environment Reference

Quick reference for local development environment, URLs, ports, and configuration.

## Local URLs

### Development Services

| Service | URL | Purpose |
|---------|-----|---------|
| **React Dev Server** | `http://localhost:3000` | UI development with hot reload |
| **Azure Function** | `http://localhost:7071` | Data connector API |
| **LucidChart Test** | `https://lucid.app/...` | Extension test environment |
| **Extension Panel** | (Embedded in LucidChart) | Loads React app via iframe |

**Access:**
- React: Auto-opens when you start Terminal 3
- Function: Test with curl or browser
- LucidChart: Manually navigate to `https://lucid.app` after Terminal 4 compiles
- Panel: Opens inside LucidChart (left sidebar)

---

## Port Mappings

### Standard Ports

| Port | Service | Protocol | Status | Notes |
|------|---------|----------|--------|-------|
| **3000** | React Dev Server | HTTP | Development only | Hot reload enabled |
| **7071** | Azure Functions | HTTP | Development only | Default local port |
| **9900** | Extension Server | HTTP | Development only | Serves extension.js |
| **9901** | Shape Libraries | HTTP | Development only | Serves shape library assets |

### Port Conflicts

If ports are in use, services won't start. See [Troubleshooting](./04_troubleshooting.md) for how to kill processes.

**Change React port:**
```bash
PORT=3001 npx react-scripts start
```

**Change Function port:**
```bash
func start --port 7072
```

**Note:** Changing ports requires updating extension config to point to new React URL

## File Locations

### Configuration Files

#### Azure Function Local Settings
**Path:** `dataconnectors/quodsi_data_connector_lucidchart_v2/local.settings.json`

**Purpose:** Local environment variables for Azure Function

**Template:** `local.settings.json.template`

**Contents:**
```json
{
  "IsEncrypted": false,
  "Values": {
    "AzureWebJobsStorage": "UseDevelopmentStorage=true",
    "FUNCTIONS_WORKER_RUNTIME": "node",
    "AZURE_STORAGE_CONNECTION_STRING": "...",
    "BATCH_ACCOUNT_URL": "...",
    "BATCH_ACCOUNT_NAME": "...",
    "BATCH_ACCOUNT_KEY": "...",
    "BATCH_POOL_ID": "dev-quodsi-pool",
    "DEFAULT_APPLICATION_ID": "dev_quodsim",
    "DEFAULT_APP_VERSION": "1.0"
  }
}
```

**Security:** In `.gitignore` - never committed

#### Extension Manifests

**Development:** `editorextensions/quodsi_editor_extension/manifest_dev.json`
**Local Testing:** `editorextensions/quodsi_editor_extension/manifest_local.json`
**Production:** `editorextensions/quodsi_editor_extension/manifest_test.json`

**Purpose:** Extension configuration for different environments

**Key fields:**
- `name`: Extension display name

- `version`: Extension version

**Development mode uses:** React dev server at `http://localhost:3000`

## Environment Variables

### Local Development

**Set in:** `local.settings.json` (Azure Function only)

**Required for simulation:**
- `AZURE_STORAGE_CONNECTION_STRING` - Azure Blob Storage
- `BATCH_ACCOUNT_URL` - Azure Batch endpoint
- `BATCH_ACCOUNT_NAME` - Batch account name
- `BATCH_ACCOUNT_KEY` - Batch account key
- `BATCH_POOL_ID` - Pool for simulation execution

**Optional:**
- `DEFAULT_APPLICATION_ID` - Batch application package (default: "dev_quodsim")
- `DEFAULT_APP_VERSION` - Application version (default: "1.0")

### Development Without Azure

You can develop without Azure credentials:
- Extension works
- React app works
- Model building/validation works
- **Simulation submission fails** (requires Azure)

For full testing, you need Azure resources.

## Log Locations

### Browser Console

**Access:** F12 → Console tab

**What's logged:**
- Extension initialization
- Message passing (extension ↔ React)
- React component lifecycle
- API calls to data connector
- Errors and warnings

**Filter logs:**
- `[SimulationHandler]` - Simulation execution
- `[MessageRouter]` - Message routing
- `[ModelPanel]` - React UI component

---

### Azure Function Logs

**Access:** Terminal 2 (where you ran `func start`)

**What's logged:**
- HTTP requests received
- Function execution start/end
- Performance metrics
- Errors and stack traces
- Batch job submission status

**Verbose mode:** Use `--verbose` flag for detailed output

---

### React Dev Server Logs

**Access:** Terminal 3 (where you ran `react-scripts start`)

**What's logged:**
- Compilation status
- Build errors/warnings
- Hot reload events
- Asset serving

---

### Extension Test Logs

**Access:** Terminal 4 (where you ran `npx lucid-package`)

**What's logged:**
- Extension build/watch status
- Webpack compilation progress
- Serving status (ports 9900/9901)
- File change notifications

## Accessing LucidChart Test Environment

### Opening the Extension

After Terminal 4 compiles successfully:

1. **Open LucidChart:** Manually navigate to `https://lucid.app` in your browser
2. **Open or create a document**
3. **In LucidChart:**
   - Look for "Quodsi" in left panel
   - Or: Click "..." (More Tools) → Find "Quodsi"

4. **Panel Opens:**
   - Loads React app from `localhost:3000`
   - Shows "Initialize Quodsi Model" if new document
   - Or shows model editor if Quodsi page

### Creating Test Documents

**Option A:** Create new document
1. Click "+ New" in LucidChart
2. Choose "Blank Document"
3. Open Quodsi panel
4. Click "Initialize Quodsi Model"

**Option B:** Use existing test document
- Test documents persist in your Lucid account
- Open from "Documents" section

### Test Data Persistence

**Local Storage:**
- Model definitions stored in LucidChart page data
- Survives page reload
- Tied to specific document

**Development Storage:**
- Shared library builds persist (until you rebuild)
- React hot reload state temporary
- Function state resets on restart

## Quick Reference Commands

### Check What's Running

**Ports in use:**
```bash
# Windows
netstat -ano | findstr "3000 7071"

# Mac/Linux
lsof -i :3000
lsof -i :7071
```

**Node processes:**
```bash
# Mac/Linux
ps aux | grep node

# Windows
tasklist | findstr node
```

---

### Rebuild Individual Components

**Shared library:**
```bash
npm run build -w @quodsi/shared
```

**Azure Function:**
```bash
cd dataconnectors/quodsi_data_connector_lucidchart_v2
npm run build
```

**React App (production build):**
```bash
cd editorextensions/quodsi_editor_extension/quodsim-react
npm run build
```

---

### Clean Build Artifacts

**Remove all build outputs:**
```bash
# From repository root
rm -rf shared/dist
rm -rf dataconnectors/quodsi_data_connector_lucidchart_v2/dist
rm -rf editorextensions/quodsi_editor_extension/quodsim-react/build
```

**Clean and rebuild:**
```bash
npm run build -w @quodsi/shared
cd dataconnectors/quodsi_data_connector_lucidchart_v2 && npm run build
```
