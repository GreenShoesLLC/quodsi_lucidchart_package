# Discovery & Browsing Experience

## Homepage

- **Featured models front and center** — hand-picked entries that showcase the registry's best content (the "Netflix hero banner").
- **Domain categories with visual tiles** — Healthcare, Manufacturing, Logistics, etc. Each shows a count of models and a few preview thumbnails.
- **Recently added or updated models.**
- **Search bar prominently placed.**

## Browsing by Domain

- Clicking a domain opens a filtered catalog view.
- Models displayed as cards — title, diagram thumbnail, summary snippet, status badge, complexity indicator.
- Sortable by: newest, most viewed, most complete, alphabetical.
- Filterable by: complexity level, status (Featured, Reviewed, Draft, Stub), has video, has implementation links.

## Search

- Full-text search across titles, summaries, and write-up content.
- Search results show the same card format for consistency.
- Domain and complexity filters available alongside search results.

## Model Entry Page

- Full content as described in [020_model_entry_structure.md](020_model_entry_structure.md), laid out for readability.
- **Scenarios** section at the top — the list of what-if questions the model is meant to answer. This is the first thing a reader sees, so they can quickly judge relevance to their own decision. See [023_scenarios.md](023_scenarios.md).
- Diagram displayed prominently below scenarios.
- Summary content (eight narrative sections) follows.
- Implementation links section — each vendor/software listed with contributor attribution.
- Model definitions section — git-backed, tagged with scenarios they support (vision).
- Related models sidebar or section at the bottom.
- Clear "Edit this entry" and "Suggest changes" calls to action for logged-in users.

## Personalization

No personalization in v1 — no favorites, no watch lists, no recommendation engine. Pure public browsing. These could come later if accounts justify the investment.
