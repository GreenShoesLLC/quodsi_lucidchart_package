# 02. Manifest Configuration

Complete reference for extension manifest files and how they control environment-specific behavior.

---

## Overview

Manifest files configure the Lucid extension package:
- Extension metadata (name, version, ID)
- OAuth providers
- **Data connector URLs** (most important for environment switching)
- Scope permissions
- Shape library locations

**Key for environment switching:** The `callbackBaseUrl` in the `dataConnectors` section determines which Azure Function the extension calls.

---

## Manifest Files

### manifest_local.json (Local Development)

**Location:** `/manifest_local.json` (repository root)

**Purpose:** Full local development with all services running locally

**Key configuration:**
```json
{
  "id": "d38c7ced-35e8-4962-a622-1d3fa480ab58",
  "version": "2024.2.65",
  "dataConnectors": [
    {
      "name": "quodsi_data_connector",
      "oauthProviderName": "lucid",
      "callbackBaseUrl": "http://localhost:7071/api/dataConnector/",
      "pollingInterval": 30
    }
  ]
}
```

**Critical field:** `callbackBaseUrl: "http://localhost:7071/api/dataConnector/"`

**Used when:**
- Running all 4 terminals locally
- Terminal 2 (Azure Function) running on port 7071
- Full debugging session

**Requires:**
- Azure Function running locally (`func start`)
- `local.settings.json` configured
- Ports 3000, 7071, 9900, 9901 available

---

### manifest_dev.json (Dev Azure Environment)

**Location:** `/manifest_dev.json` (repository root)

**Purpose:** Testing extension against dev Azure backend

**Key configuration:**
```json
{
  "id": "8dc6bc70-c635-4263-a104-179c1412d979",
  "version": "2024.2.60",
  "dataConnectors": [
    {
      "name": "quodsi_data_connector",
      "oauthProviderName": "lucid",
      "callbackBaseUrl": "https://dev-quodsi-func-v1.azurewebsites.net/api/dataConnector/",
      "pollingInterval": 30
    }
  ]
}
```

**Critical field:** `callbackBaseUrl: "https://dev-quodsi-func-v1.azurewebsites.net/api/dataConnector/"`

**Used when:**
- Testing with dev backend
- Don't want to run local Azure Function
- Integration testing

**Requires:**
- Dev Azure Function deployed and running
- No local Terminal 2 needed
- React `.env.production` should match this URL

**Note:** Different extension ID from local (allows both to be installed simultaneously)

## Key Manifest Fields

### id

**Purpose:** Unique identifier for the extension package

**Format:** UUID

**Values:**
- `manifest_local.json`: `d38c7ced-35e8-4962-a622-1d3fa480ab58`
- `manifest_dev.json`: `8dc6bc70-c635-4263-a104-179c1412d979` (different!)
- `manifest_test.json`: `d38c7ced-35e8-4962-a622-1d3fa480ab58` (same as local)

**Why different?**
- Allows installing both local and dev versions simultaneously in Lucid
- Local and test share ID (you'd only use one at a time)
- Dev has unique ID (can coexist with local)

---

### version

**Purpose:** Extension version number

**Format:** Semantic versioning (YYYY.M.MINOR)

**Current:**
- `manifest_local.json`: `2024.2.65`
- `manifest_dev.json`: `2024.2.60`
- `manifest_test.json`: `2024.2.60`

**Impact:** Lucid uses this to determine if extension needs updating

---

### dataConnectors[].callbackBaseUrl

**Purpose:** Base URL for Azure Function data connector

**Format:** URL with `/api/dataConnector/` path

**THIS IS THE MOST IMPORTANT FIELD FOR ENVIRONMENT SWITCHING**

**Values:**
- Local: `http://localhost:7071/api/dataConnector/`
- Dev: `https://dev-quodsi-func-v1.azurewebsites.net/api/dataConnector/`
- Test: `https://tst-quodsi-func-v1.azurewebsites.net/api/dataConnector/`
- Prod: `https://prd-quodsi-func-v1.azurewebsites.net/api/dataConnector/`

**How it's used:**
```typescript
// Lucid SDK automatically appends action names:
// callbackBaseUrl + action name
// Examples:
// http://localhost:7071/api/dataConnector/saveAndSubmitSimulationAction
// http://localhost:7071/api/dataConnector/poll
```

**Must match:** React app's `REACT_APP_DATA_CONNECTOR_API_URL` (minus `/dataConnector/` path)

---

### dataConnectors[].pollingInterval

**Purpose:** How often (in seconds) to poll for data connector updates

**Value:** `30` (30 seconds)

**Impact:** Frequency of status updates for simulations

---

### oauthProviders

**Purpose:** Configure OAuth integration with Lucid API

**Configuration:**
```json
{
  "name": "lucid",
  "title": "Lucid",
  "authorizationUrl": "https://lucid.app/oauth2/authorize",
  "tokenUrl": "https://api.lucid.co/oauth2/token",
  "scopes": ["lucidchart.document.content:readonly"],
  "domainWhitelist": ["https://api.lucid.co"],
  "clientAuthentication": "clientParameters"
}
```

**Same across all manifests** - OAuth endpoints don't change per environment

**Why needed:** LucidDataActionUtility OAuth workaround for accessing Lucid API

---

### scopes

**Purpose:** Permissions required by extension

**Values:**
```json
[
  "READ",
  "WRITE",
  "DOWNLOAD",
  "SHOW_MODAL",
  "CUSTOM_UI",
  "NETWORK",
  "USER_INFO"
]
```

**Same across all manifests**

## How Lucid Package Tool Uses Manifests

### Test Mode (npx lucid-package test-editor-extension)

**Command:**
```bash
npx lucid-package@latest test-editor-extension quodsi_editor_extension
```

**Manifest selection:**
- Looks for `manifest.json` in current directory
- Uses that as configuration
- **Does NOT automatically pick manifest_local.json**

**Workaround:** Copy desired manifest before testing:
```bash
# Use local manifest
cp manifest_local.json manifest.json

# Run test
npx lucid-package@latest test-editor-extension quodsi_editor_extension
```

**Or:**
```bash
# Specify manifest explicitly (if supported by lucid-package version)
npx lucid-package@latest test-editor-extension quodsi_editor_extension --manifest manifest_local.json
```

---

### Deployment Mode

**For dev deployment:**
```bash
# Copy dev manifest

# Build and deploy
# ... deployment scripts ...
```

**For test deployment:**
```bash
# Copy test manifest
cp manifest_test.json manifest.json

# Build and deploy
# ... deployment scripts ...
```

## Environment Switching

### Switching from Local to Dev Azure

**Scenario:** You've been running locally, now want to test against dev Azure

**Steps:**

1. **Copy manifest:**
   ```bash
   cp manifest_dev.json manifest.json
   ```

2. **Stop local Azure Function:**
   - Ctrl+C in Terminal 2 (no longer needed)

3. **Verify React .env:**
   - Check `.env.production` has: `REACT_APP_DATA_CONNECTOR_API_URL=https://dev-quodsi-func-v1.azurewebsites.net/api/`

4. **Rebuild React (if needed):**
   ```bash
   cd editorextensions/quodsi_editor_extension/quodsim-react
   npm run build
   ```

5. **Restart extension:**
   - Ctrl+C in Terminal 4
   - Rerun: `npx lucid-package@latest test-editor-extension quodsi_editor_extension`

6. **Refresh browser:**
   - Hard refresh (Ctrl+Shift+R)
   - Clear sessionStorage if auth issues

---

### Switching from Dev Azure to Local

**Scenario:** You've been testing against dev Azure, now want full local debugging

**Steps:**

1. **Copy manifest:**
   ```bash
   cp manifest_local.json manifest.json
   ```

2. **Start local Azure Function:**
   ```bash
   cd dataconnectors/quodsi_data_connector_lucidchart_v2
   npm run build
   func start --debug --verbose
   ```

3. **Verify local.settings.json exists**

4. **Restart extension:**
   - Ctrl+C in Terminal 4
   - Rerun: `npx lucid-package@latest test-editor-extension quodsi_editor_extension`

5. **Refresh browser:**
   - Hard refresh (Ctrl+Shift+R)

## Common Issues

### Issue: API Calls Going to Wrong Backend

**Symptoms:**
- Extension calls localhost but should call Azure
- Extension calls dev Azure but should call localhost
- CORS errors
- 404 or connection refused errors

**Cause:** Mismatch between manifest's `callbackBaseUrl` and actual running backend

**Debug:**
```bash
# Check current manifest
cat manifest.json | grep callbackBaseUrl

# Check if local function running
curl http://localhost:7071

# Check extension manifest in use
# Browser console: Look for data connector calls in Network tab
```

**Fix:**
1. Copy correct manifest: `cp manifest_local.json manifest.json`
2. Ensure backend is running (Terminal 2 for local, or Azure for dev/test)
3. Restart Terminal 4 (extension)
4. Hard refresh browser

---

### Issue: Extension Not Picking Up Manifest Changes

**Symptoms:**
- Changed manifest.json but extension still uses old callbackBaseUrl
- Network tab shows wrong URL

**Cause:** Extension loaded old manifest into memory

**Fix:**
1. Stop Terminal 4 (Ctrl+C)
2. Verify manifest.json has correct values: `cat manifest.json | grep callbackBaseUrl`
3. Restart Terminal 4: `npx lucid-package@latest test-editor-extension quodsi_editor_extension`
4. Wait for "webpack compiled successfully"
5. Hard refresh browser (Ctrl+Shift+R)
6. Check Network tab for correct URL

---

### Issue: Both Local and Dev Extensions Installed

**Symptoms:**
- Two "Quodsi" entries in left panel
- Confusion about which is which

**Cause:** Different extension IDs allow both to be installed

**Solution:**
- Uninstall one from Lucid account settings
- Or: Use different Lucid accounts for local vs dev testing

**Check which is which:**
```bash
# Local ID
grep '"id"' manifest_local.json

# Dev ID
grep '"id"' manifest_dev.json
```

---

### Issue: React App URL Mismatch

**Symptoms:**
- Extension loads but React panel blank
- Console shows failed to load resources from wrong URL

**Cause:** Manifest specifies React app URL (if using custom URL)

**Note:** This project doesn't specify React URL in manifest (loaded via localhost:3000 automatically in test mode)

**If issue persists:** Check browser console for actual URL being loaded

## Manifest and React .env Compatibility

**CRITICAL:** Manifest `callbackBaseUrl` and React `REACT_APP_DATA_CONNECTOR_API_URL` must match!

### Correct Combinations

**Local development:**
```json
// manifest_local.json
"callbackBaseUrl": "http://localhost:7071/api/dataConnector/"
```
```bash
# .env.local or default
REACT_APP_DATA_CONNECTOR_API_URL=http://localhost:7071/api/
```

**Dev Azure:**
```json
// manifest_dev.json
"callbackBaseUrl": "https://dev-quodsi-func-v1.azurewebsites.net/api/dataConnector/"
```
```bash
# .env.production
REACT_APP_DATA_CONNECTOR_API_URL=https://dev-quodsi-func-v1.azurewebsites.net/api/
```

**Note:** React URL doesn't have `/dataConnector/` path - that's appended by Lucid SDK in manifest usage

## Best Practices

### DO

✅ Keep manifest.json in .gitignore (it's environment-specific)
✅ Copy manifest before testing: `cp manifest_local.json manifest.json`
✅ Verify callbackBaseUrl matches your backend
✅ Restart Terminal 4 after manifest changes
✅ Document which manifest you're using in comments/notes

### DON'T

❌ Commit manifest.json to git (should be generated/copied)
❌ Edit manifest.json directly (edit specific manifest_*.json instead)
❌ Forget to restart extension after manifest change
❌ Mix local manifest with Azure backend (or vice versa)

## Checklist: "Which Manifest Should I Use?"

**Use manifest_local.json when:**
- [ ] Running all services locally (4 terminals)
- [ ] Terminal 2 (Azure Function) running on port 7071
- [ ] Debugging full stack
- [ ] Need to modify data connector code

**Use manifest_dev.json when:**
- [ ] Testing against dev Azure backend
- [ ] Don't need to debug data connector
- [ ] Want to test integration with deployed backend
- [ ] Data connector changes already deployed to dev

- [ ] Pre-production QA testing
- [ ] Validating before production deploy
- [ ] Testing with test Azure environment
